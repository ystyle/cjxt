# 介绍

## cjxt 是什么？

cjxt 是一个基于**仓颉语言**的全栈服务端驱动 UI 框架。使用仓颉编写完整的 Web 应用——从路由、状态管理到 UI 渲染，全在服务端完成。

## 核心优势

### 服务端驱动，状态统一

传统 Web 开发中，前端需要维护一套状态，后端需要维护另一套，靠 API 契约连接。cjxt 把状态统一放在服务端，前端只做轻量渲染，不再需要前端状态管理、API 请求封装、数据同步等复杂工作。

### 零前端框架

前端 JS 不到 5KB，无 React、Vue、Angular 等依赖。通过 WebSocket 接收服务端推送的 JSON 增量更新，直接操作 DOM。

### 全栈类型安全

仓颉的类型系统贯穿前后端——Signal 的泛型参数、ActionContext 的参数、组件的 Props，在编译期即可检查，无需手写类型定义文件或 Schema。

### 响应式更新

Signal 提供自动依赖追踪，组件只重渲变化的部分。不需要 Virtual DOM diff，不需要手动标记更新范围。

### Element Plus 设计语言

组件库的 Props、Events、样式对齐 Element Plus，SCSS 源文件直接复用 EP 官方变量。对 Element Plus 开发者来说，上手成本极低。

## 适用场景

- **中后台管理系统**：表单、表格、复杂交互，服务端驱动天然适合
- **全栈应用**：从数据库到 UI 全部仓颉，不需要两套语言和框架
- **内网工具**：对 SEO 不敏感，但对开发效率和类型安全要求高的场景
- **SSR 优先**：天然服务端渲染，无需额外配置

## 前提

- 熟悉仓颉语言基础语法
- 了解 Element Plus 的基本组件概念
- 不要求前端框架经验
